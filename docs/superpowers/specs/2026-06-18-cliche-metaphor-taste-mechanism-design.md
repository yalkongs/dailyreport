# 클리셰·은유 메커니즘 — 규칙(rules) → 취향(taste) 전환

작성일: 2026-06-18
상태: 설계 승인됨 (구현 계획 대기)
분류: "리포트 문장 자연스러움 개선"의 **sub-project A** (B 헤드라인 재설계 · C 컴플라이언스 이동 · D 문장 운율 완화는 각자 별 사이클)

## 문제

두 리포트(마켓·ETF)의 문장·제목이 부자연스러운 경우가 많다. 진단 결과 이는
**AI(모델)의 한계가 아니라, 시스템이 취향(taste)을 무딘 규칙으로 대리**하면서
생긴 부산물이다. 같은 모델이 규칙이 가로채지 않은 자리에서는 자연스럽고 신선한
문장을 쓴다(아래 "살아남은 자리" 참조).

이 sub-project는 그중 **클리셰·은유 메커니즘** 한 조각만 다룬다.

### 실증된 두 가지 실패 모드

1. **금지어 리스트의 사각 — 구조 클리셰를 못 막음.** 클리셰 단일 소스
   `lib/banned-metaphors.ts`는 **단어**(파도·항해·등대…)만 금지한다. 그래서 모델이
   리스트에 없는 "안전한" **구문 틀**을 찾아 반복했다 — 최근 마켓 헤드라인 12개 중
   **5개**가 "X가 그린 [지도/로드맵/고속도로]"였다(06-08·09·10·12·15). 금지된
   단어들보다 더 큰 클리셰가 다른 문으로 들어왔다.
2. **과잉 억제 — 좋은 은유까지 죽임.** 단어 금지는 식상한 비유와 신선한 비유를
   구분하지 못해 모델을 비유 자체에 위축시킨다. 사람은 "파도는 식상하나 *이*
   이미지는 신선하다"를 판단해 피하지만, 리스트엔 판단이 없어 그냥 금지한다.

### 살아남은 자리 (= 모델은 할 수 있다는 증거)

규칙이 가로채지 않은 자리의 실제 출력:
- 마켓 헤드라인: "연준이 금리를 올린 밤, 달러는 올랐고 나머지는 내려앉았다"(06-18),
  "원유가 내려앉은 자리에 주가가 올라섰다"(06-16)
- ETF 헤드라인: "반도체가 빠진 자리, 금과 건설이 채운 하루"(06-17)
- 본문: "주식과 채권이 같은 날 함께 내려앉은 밤이었습니다"(마켓), "괴리율 확대가
  체결 비용을 조용히 갉아먹습니다"(ETF)

자연스럽고·신선하고·클리셰 아니고·정확하다. 셋이 동시에 됐다.

## 목표 (성공 기준)

규칙(금지/필터)을 **취향(예시·원칙·판단)**으로 바꿔, **클리셰 방지를 유지하면서
자연스러움과 신선한 은유를 동시에** 얻는다. 구체 판정:
- 구조 템플릿 반복("X가 그린 Y" 류)이 사라진다.
- 은유가 살아있되 신선하다(매번 붙일 수 있는 비유 아님).
- 알려진 죽은 클리셰(파도·항해 등)가 재발하지 않는다.

이 방향은 기존 사용자 지침과 정렬한다: [[feedback-headline-metaphor-tone]](은유
과감히 허용·과도 제약 금지), [[feedback-prompt-consolidation]](모순 패치 누적 말고
통합).

## 비목표

- 헤드라인 앵커/tier 재설계(sub-project B), 컴플라이언스 정규식 이동(C),
  문장 운율/명사끝 금지 완화(D) — **이 스펙 범위 밖.**

  (로그 기록 제외 필터) 통합 안 하기로 한 기존 결정을 존중. **불변.**
- 연산 파이프라인 구조·렌더러 변경. 이 작업은 **프롬프트 콘텐츠 + 예시 모듈**이
  거의 전부다.

## 설계

### 1. 변경 지점 (배선)

- **신규 단일 소스 `lib/voice-exemplars.ts`** — 목표 보이스 예시. `banned-metaphors.ts`가
  클리셰 단일 소스이듯, 예시의 단일 소스. 마켓·ETF 공유.
- **마켓 프롬프트 `lib/claude-client.ts:58`** — "❌ 죽은 클리셰 금지: ${전체 리스트}
  등" → 긍정 원칙 + 소프트 칼리브레이션(§3)으로 교체.
- **마켓 `lib/claude-client.ts:447`** — "클리셰는 금지(목록)" 참조를 원칙 참조로 갱신.
- **ETF 프롬프트 `lib/etf/claude-client.ts:39`** — 이미 "신선·구체 은유 권장"이라 결이
  맞음. 예시 블록 추가 + 문구를 §3 원칙과 정렬.
- **`sanitizeBannedExpressions` (`claude-client.ts:595`)** — 이미 **경고 로그만**
  ("제거하지 않고 경고만"). 하드 게이트 아님 → **관측용으로 유지, 손대지 않음.**
- **`narrative-memory.ts:116`** — 불변(비목표).

> 발견: 마켓 anti-repetition 블록(`:95-118`)은 **이미 최근 헤드라인·은유를 "반복
> 금지"로 먹인다.** "그린 지도"가 반복된 건, 모델이 *정확히 같은 제목*만 피하고
> **구조를 추상화하지 못해서**다. 따라서 §4는 새 시스템이 아니라 이 블록의 **지시
> 강화**다.

### 2. 예시 모듈 `lib/voice-exemplars.ts` (이 작업의 심장)

자사 살아남은 문장을 **시드**로 큐레이션:
- 헤드라인 시드: "연준이 금리를 올린 밤, 달러는 올랐고 나머지는 내려앉았다",
  "원유가 내려앉은 자리에 주가가 올라섰다", "반도체가 빠진 자리, 금과 건설이 채운 하루"
- 본문 시드: "주식과 채권이 같은 날 함께 내려앉은 밤이었습니다", "괴리율 확대가
  체결 비용을 조용히 갉아먹습니다"

**사용자가 이 시드를 검토·편집·추가**해 천장을 설정한다 — **구현 단계의 협업
포인트**(나는 시드 제안, 사용자가 확정/보강).

(선택) 반례 예시도 같은 모듈에 둘 수 있다: 구조 클리셰 "X가 그린 [지도]", 데이터피드
제목 "SOXX +1.44%, …" — "이렇게는 쓰지 말 것"의 구체 앵커.

### 3. 원칙 문구 (부정 규칙 → 긍정 + 소프트 칼리브레이션)

금지 리스트 나열을 다음으로 대체(양 프롬프트 공통):

> "데이터에 진실한 구체적 이미지 하나가 제 몫을 하면 환영한다. **매번 어느 날에나
> 붙일 수 있는** 진부한 비유와 **반복되는 구문 틀**은 피하라(진부함이란 이런 것:
> 파도·항해·롤러코스터·폭풍 등). 문장·제목의 형태를 변주하라."

핵심: 금지 리스트가 **하드 필터에서 '진부함이 무엇인지' 가르치는 짧은 예시로
강등**된다.

### 4. anti-repetition 강화 (`lib/claude-client.ts:95-118`)

- 현재: 최근 헤드라인을 "반복 금지"로 나열 → 모델이 정확히 같은 제목만 회피.
- 변경: 지시를 **"단어가 아니라 구조·이미지·구문 틀을 재사용하지 말 것"**으로 명시.
  최근 헤드라인에 더해 **서두 문장**도 함께 제시하고, "최근 제목들의 공통 패턴
  (예: 'X가 그린 Y')을 감지하면 다른 골격으로 써라"를 추가.
- 프롬프트 레벨만. 하드 게이트·재생성 추가 없음.

### 5. 범위·검증

- **범위**: 마켓·ETF **양쪽**, 헤드라인 + 본문 서사 필드.
- **검증**: "자연스러움"은 단위테스트 불가. **백필 비교** — 전/후로 N개(예: 5~10)
  리포트를 생성해 헤드라인·서브라인·본문을 눈으로 대조: ① 구조 템플릿 반복 사라짐
  ② 은유 살아있되 신선 ③ 죽은 클리셰 재발 없음. 기존 단위 테스트는 green 유지.
- **트레이드오프(문서화)**: 하드 필터를 제거하므로 첫-출현 클리셰를 가끔 놓칠 수
  있다. 예시+원칙+소프트 칼리브레이션이 이를 줄이되 100% 보장은 아니다. 백필에서
  재발 빈도를 관찰해 필요 시 칼리브레이션 예시를 보강(리스트 부활 아님).

## 구현 작업 성격

거의 전부 **프롬프트 콘텐츠 편집 + 신규 예시 모듈**. 연산/렌더러 코드 변경 없음.
동작(생성 결과)이 바뀌므로 백필 검증이 필수. 예시 큐레이션은 사용자 협업 단계.
