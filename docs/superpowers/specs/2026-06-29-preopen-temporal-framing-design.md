# 개장 전 시점 프레이밍 가드 — 설계

> 2026-06-29 인시던트(월요일 market 리포트가 금요일 종가를 "오늘 폭락 마감"으로 서술 + 주말 뉴스 인과 날조)의 근본 수정.
> 이 스펙은 캘린더 감사(`2026-06-02-calendar-timezone-audit-findings.md`)의 **sub-project B(휴일 인지 서사 가드레일, 미착수)** 를 재정의한 일부다.

## 1. 배경 — 무엇이 어떻게 깨졌나

2026-06-29(월) 06:43 KST 발행된 market 리포트(`/reports/2026-06-29`)가:
- 코스피 8,411 / −5.81%(**금요일 6/26 실제 종가**, 데이터는 정상)를 **"오늘 월요일 서울 장이 열리자마자 매도가 쏟아져 마감했다"** 로 서술.
- **"주말 사이 미·이란 충돌이 오늘 폭락을 유발"** 이라는 시간상 불가능한 인과를 날조(금요일 종가가 주말 뉴스에 반응 불가).

발행 시각은 **한국 장 개장(09:00) 전**이라 월요일 KR 세션은 시작도 안 했다.

### 근본 원인 2겹

**① 시점 가드 부재 — `lib/claude-client.ts:285`**
```js
const calendarBlock = (() => {
  const info = ctx.calendarInfo;
  if (!info || (!info.isKrClosedOnly && !info.isUsClosedOnly)) return "";  // ← 정상 거래일엔 빈 문자열
  ...
})();
```
"이 지수는 직전 영업일 종가다, '오늘 마감'으로 쓰지 말라"는 가드가 **휴장 분기(한국 휴장/미국 휴장)에만** 존재. 양국 정상일이면 빈 문자열 → 가드 0. 그러나 리포트는 **항상 06:40 한국 개장 전 생성**(프롬프트 `:447`도 "배달 시점=한국 개장 전" 명시)이라 KR 지수는 **늘 직전 종가**. 코드가 "오늘이 거래일인가"와 "오늘 장이 이미 끝났나"를 혼동 — 06:40엔 안 끝났다. **매 평일 잠복.**

**② 월요일 리듬이 날조를 트리거 — `lib/weekday-rhythm.ts:34-38`**
무조건 주입(`:327`)되는 월요일 분기:
> "주말 사이 발생한 해외 뉴스·정책 변화를 본문 앞부분에서 한 단락으로 정리… 헤드라인은 '주말 뒤 시작'의 의미가 묻어나도록"

①의 가드가 없는 상태에서 이 지시가 결합 → 금요일 종가를 오늘 폭락으로 오인 + 주말 뉴스를 그 원인으로 서술 → 인과 날조.

### 왜 지금까지 안 띄었나
Systemic — 6/26 리포트도 목요일 종가를 "한 주를 닫는 오늘"로 프레이밍. 평일은 1일 갭 + 평온하면 미묘. **월요일은 3일 갭 + 고변동성 + 날조 인과**가 겹쳐 폭발.

### Market vs ETF 차이 (핵심)
ETF 시스템 프롬프트(`etf/claude-client.ts:227-234`)는 "발행=06:30 개장 전, 주된 데이터=간밤 해외+전일 국내"를 **상시 명시**해 같은 calendarBlock 조기반환(`:190`)을 공유함에도 모델이 보호된다. **Market엔 이 베이스라인 프레이밍이 없다** — 그게 근본 차이.

## 2. 목표 / 비목표

**목표 (이번 스펙)**
- KR 지수(매 평일) + US 지수(월요일 갭)의 시점 오라벨링 근본 해결.
- 지수 데이터에 **실제 세션 날짜 + 갭 인지(간밤 / 지난 금요일)** 를 **항상** 라벨링.
- 월요일 리듬을 가드와 양립하게(주말 뉴스 = "개장 시 반영될 이번 주 변수", "오늘 움직임의 원인" 아님).
- ETF는 가벼운 일관 적용(이미 안전, 월요일 갭만 보강) — 2차 우선.

**비목표 (sub-project B 잔여로 유지·deferred)**
- 휴장 엣지케이스: H2(US 하루어긋남)·M1(KR 대칭 갭)·M2(EU/JP/CN 휴일 캘린더). 휴장 분기는 **현행 유지**, 손대지 않음.
- 24h 상품 라벨(M3)·marketState 해시(M4)·운영자 알림(M5/F). 별도.

## 3. 설계 — 상시 갭 인지 시점 블록

### 3.1 갭 인지 헬퍼 (신규, 결정론·테스트 가능)
`lib/market-calendar.ts`에 추가:
```ts
/** 리포트 날짜와 직전 영업일 사이 '시점 표현' 결정.
 *  gap=1(어제) → 간밤/전 거래일, gap>1(주말·연휴) → 실제 요일 명시. */
export function describeSessionRecency(
  reportDate: string,        // "2026-06-29"
  prevTradingDay: string,    // "2026-06-26"
  market: "kr" | "us"
): { gapDays: number; phrase: string; weekday: string }
```
- `gapDays` = `reportDate`와 `prevTradingDay`의 **달력일 차이**.
- `phrase`:
  - `gapDays === 1`: KR → `"전 거래일(어제, {date})"`, US → `"간밤({date} 현지 마감)"`.
  - `gapDays > 1`: 양쪽 → `"지난 {weekday}({date})"` (예: "지난 금요일(2026-06-26)"). **"간밤"·"오늘"·"주말 뒤 첫 장" 같은 표현 금지 신호 포함.**
- `weekday` = `prevTradingDay`의 한국어 요일.

### 3.2 market 상시 시점 블록 — `lib/claude-client.ts`
`calendarBlock`을 **항상 발화하는 `temporalFramingBlock`** 으로 대체:
- **휴장 분기(`isKrClosedOnly`/`isUsClosedOnly`)는 현행 텍스트 보존** (비목표 영역).
- **신설 정상일 분기**(양국 open)에서 다음을 명시:
  1. "이 리포트는 **{date}({요일}) 한국 장 개장(09:00) 전** 06:40에 작성된 **개장 전 브리핑**입니다."
  2. KR 지수(코스피·코스닥·원/달러): `describeSessionRecency(date, krPrevTradingDay, "kr")`의 phrase로 "**{phrase} 종가 기준**" 명시. **하드룰: "오늘 코스피가 X로 마감했다" 금지** — 오늘 KR 세션 미발생.
  3. US 지수(S&P·나스닥·다우·VIX·미 10Y): `describeSessionRecency(date, usPrevTradingDay, "us")` phrase로 명시.
  4. **gap>1(월요일 등)** 일 때: "직전 거래일과 오늘 사이 **{gapDays-1}일의 비거래일**이 있다. 그 사이 뉴스는 **'오늘 개장 시 반영될 변수'** 로 서술하되, **직전 종가가 이미 반영한 것처럼 쓰지 말 것.**"

### 3.3 월요일 리듬 재구성 — `lib/weekday-rhythm.ts:34-38`
market 변형(`reportType==="market"`)의 월요일 텍스트를 전방위로:
- 현행: "주말 사이 해외 뉴스를 본문 앞 단락에 정리… 헤드라인은 '주말 뒤 시작'"
- 변경: "직전 영업일(지난 금요일) 종가가 기준이며 **오늘 장은 미개장**이다. 주말 사이 해외 뉴스는 **'오늘 개장 시 주목할 변수 / 이번 주 관전 포인트'** 로 전방위 서술 — **오늘 이미 일어난 움직임의 원인으로 쓰지 말 것.**"
- ETF 변형은 overnight 브리핑 설계와 정합하므로 **현행 유지**(주말 해외=주된 데이터가 맞음).

### 3.4 ETF 일관 적용 (2차) — `lib/etf/claude-client.ts:190`
ETF는 베이스라인이 안전하나, 월요일 US "간밤" 갭만 보강: 정상일 분기에서 US 데이터에 `describeSessionRecency(date, usPrevTradingDay, "us")` phrase 적용(gap>1이면 "간밤"→"지난 금요일 밤"). KR은 ETF 프롬프트가 이미 "전일"로 처리하므로 최소 변경.

## 4. 데이터 흐름
`run.ts`/`run-etf.ts`가 `getMarketCalendarInfo(date)` → `calendarInfo`(이미 `krPrevTradingDay`/`usPrevTradingDay` 보유) → 프롬프트 빌더 `ctx.calendarInfo`로 전달(현행 그대로). 신규 헬퍼는 이 필드만 소비 — **새 데이터 수집 없음.**

## 5. 테스트 전략 (이전 '자연스러움 단위테스트 불가' 공백을 메움)
**결정론적 단위 테스트** — `lib/market-calendar.test.ts`에 `describeSessionRecency` 케이스:
- 화요일(2026-06-30, prev=06-29): KR `gapDays===1`·phrase에 "전 거래일"·"간밤" 없음(KR); US phrase에 "간밤".
- **월요일(2026-06-29, prev=06-26)**: 양쪽 `gapDays===3`·phrase에 "지난 금요일"·"간밤" **불포함**.
- 연휴 뒤(가용 시): `gapDays>1` 동작.

프롬프트 블록 생성기(정상일 분기)도 가능하면 문자열 단위 검증(월요일 calendarInfo 주입 시 "지난 금요일" 포함, "오늘 코스피" 류 금지 문구 포함).

**백필 검증** — 오늘 6/29 스냅샷으로 market 리포트 재생성 → "오늘 ... 마감"·"주말 뒤 첫 장"·날조 인과가 사라지고 "지난 금요일 종가 기준"으로 바뀌는지 육안 비교.

- 백필 비교 1회(6/29 데이터).

## 7. 리스크
- 프롬프트에 시점 제약을 강하게 넣으면 문장이 경직될 수 있음. 단 **시점 정확성은 취향이 아니라 사실** — sub-project A(클리셰→취향)와 충돌 아님. 헤드라인 톤 자유(`feedback_headline_metaphor_tone`)는 유지하되 "오늘 마감" 류 **사실 오류만** 차단.
- ETF 변경은 안전한 코드를 건드리므로 최소·2차로 한정, 백필로 회귀 확인.
